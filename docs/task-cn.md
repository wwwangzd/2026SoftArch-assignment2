# 属性驱动设计（Attribute-Driven Design，ADD）方法

## 第 1 步：审查输入

ADD 方法的第一步是审查输入，并识别哪些需求将被视为体系结构驱动因素。

## 第 2 步：通过选择驱动因素来确立迭代目标

一轮设计通常表现为一系列设计迭代，其中每次迭代都专注于实现一个特定目标。这样的目标通常涉及通过设计来满足一部分驱动因素。

## 第 3 步：选择一个或多个需要细化的系统元素

这一步是核心设计活动开始的地方。你将选择的元素，是那些参与满足特定驱动因素的元素。对于绿地开发，可以先建立系统上下文，然后选择唯一可用的元素，也就是系统本身，通过分解来进行细化。对于现有系统，或绿地系统后续设计迭代中的情况，通常会选择细化在先前迭代中识别出的元素。

## 第 4 步：选择一个或多个能够满足所选驱动因素的设计概念

这一步要求你识别可用于实现迭代目标的设计概念备选方案，并从这些备选方案中选择一个。

## 第 5 步：实例化体系结构元素、分配职责并定义接口

这一步要求基于所选设计概念实例化体系结构元素，并为它们分配职责。此外，还需要建立元素之间的关系和接口，使它们能够有效协作并交换信息。

## 第 6 步：绘制视图草图并记录设计决策

此时，你已经完成了该次迭代的设计活动。除了保留视图，也就是你所创建结构的表示之外，还应记录设计迭代中做出的重要决策，以及这些决策背后的原因（即理由），以便后续分析和理解。

## 第 7 步：分析当前设计，并审查迭代目标与设计目的的达成情况

这一步检查是否已经创建出满足当前迭代目标的局部设计，并考虑是否还需要额外的设计迭代。

# 使用 ADD 设计成熟领域中的绿地系统：酒店定价系统

## 设计目的

该项目可以视为绿地开发，因为它涉及对现有系统的完全替换。设计活动的目的是做出初始决策，以支持从零开始构建系统。

## 主要功能

| 用例 | 描述 |
| --- | --- |
| HPS-1：登录 | 用户（商务用户或管理员）在登录窗口中提供凭据。系统通过用户身份服务检查这些凭据；如果检查成功，则允许访问系统。登录后，用户只能查询和修改其被授权管理的酒店。 |
| HPS-2：修改价格 | 用户选择一个其被授权修改价格的特定酒店，并选择想要修改价格的日期，可以修改基础价格或固定价格。所有由基础价格计算得到的费率价格都将在此时计算出来。系统允许在实际修改前模拟价格变化。当价格被修改后，这些价格会被推送到渠道管理系统，并可供外部系统查询。 |
| HPS-3：查询价格 | 用户或外部系统通过用户界面或查询 API 查询给定酒店的价格。 |
| HPS-4：管理酒店 | 管理员添加、变更或修改酒店信息。这包括编辑酒店的税率、可用费率和房型。 |
| HPS-5：管理费率 | 管理员添加、变更或修改费率。这包括定义不同费率的计算业务规则。 |
| HPS-6：管理用户 | 管理员修改给定用户的权限。 |

## 质量属性

| ID | 质量属性 | 场景 | 关联用例 | 对客户的重要性 | 实现难度 |
| --- | --- | --- | --- | --- | --- |
| QA-1 | 性能 | 在正常运行期间，某个酒店某个日期的基础价格发生改变；该酒店所有费率和房型的价格在少于 100 ms 内发布（可供查询）。 | HPS-2 | 高 | 高 |
| QA-2 | 可靠性 | 用户对给定酒店执行多次价格修改；100% 的价格修改都成功发布（可供查询），并且也被渠道管理系统接收。 | HPS-2 | 高 | 高 |
| QA-3 | 可用性 | 除维护窗口外，价格查询的正常运行时间 SLA 必须达到 99.9%。 | 全部 | 高 | 高 |
| QA-4 | 可扩展性 | 系统初始阶段将通过 API 每天至少支持 100,000 次价格查询，并且应能够处理最多 1,000,000 次查询，同时平均延迟性能劣化不超过 20%。 | HPS-3 | 高 | 高 |
| QA-5 | 安全性 | 用户通过前端登录系统。用户凭据会通过用户身份服务进行验证；登录后，用户只会看到其被授权使用的功能。 | 全部 | 高 | 中 |
| QA-6 | 可修改性 | 为系统添加一个与 REST 不同协议的价格查询端点（例如 gRPC）。新的端点不需要修改系统核心组件。 | 全部 | 中 | 中 |
| QA-7 | 可部署性 | 在开发过程中，应用程序会在非生产环境之间迁移。迁移时不需要修改代码。 | 全部 | 中 | 中 |
| QA-8 | 可监控性 | 系统运维人员希望在运行期间度量价格发布的性能和可靠性。系统提供一种机制，允许按需收集 100% 的这些度量。 | HPS-2 | 中 | 中 |
| QA-9 | 可测试性 | 系统及其元素应 100% 支持在独立于外部系统的情况下进行集成测试。 | 全部 | 中 | 中 |
